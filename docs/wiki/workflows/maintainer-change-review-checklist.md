# 宫寒项目维护者审核清单

本文是 Gonghan-Princess 维护的 `AI-Novel-Writing-Assistant` 内部变更验收流程，用于合并前自查、并发任务交接和维护者复核。它不再引导到上游元项目或其他仓库提交 PR；所有结论都以本仓库当前目标、文档口径和用户体验为准。

## 适用范围

当变更影响公开文档、用户界面、后端工作流、AI 生成内容、第三方材料、测试结果或发布说明时，维护者应按本文检查。纯本地试验、临时草稿或未进入项目主线的探索可以不走完整清单，但一旦准备保留到仓库，就需要补齐对应记录。

## 审核原则

- 先确认变更服务的是宫寒个人版小说工作台，而不是上游元项目、通用模板或无关 fork。
- 先看用户可见路径，再看实现细节：入口、文案、状态、失败恢复和长期维护成本都要能解释清楚。
- 并发开发时顺着当前文件现状调整，不回滚别人已经写入的改动。
- AI 辅助生成、第三方材料和测试结果必须可追溯；无法验证的地方要明确说明原因和剩余风险。

## UI 与交互审核

- 入口是否仍然围绕宫寒创作驾驶舱、自动导演、章节生产、知识库、拆书、漫画或短剧等本项目主流程。
- 文案是否面向 Gonghan-Princess 项目自身，不出现“回到元项目提交 PR”“同步上游评审”等误导。
- 新增按钮、菜单、页面或说明是否能被第一次使用者理解，并且不会把维护者入口混进普通写作主流程。
- 关键状态是否可见：加载中、成功、失败、可重试、需人工确认、已跳过。
- 视觉变更是否和当前宫寒个人版品牌一致；截图、公开介绍和 README 如受影响，应同步更新。

## 后端与数据审核

- API、任务、数据库状态或后台队列是否仍由本项目现有边界承接，避免新增绕过既有 Runtime、Service、Prompt Registry 或任务状态投影的捷径。
- 失败路径是否有清晰返回：错误信息、重试条件、日志位置、是否会重复扣费或重复写入。
- 迁移、脚本或配置变更是否说明默认值、兼容旧数据的方式和回滚思路。
- 涉及文件、图片、模型输出或知识库索引时，检查落盘路径、清理策略和权限边界。
- 若只改文档，应确认没有夹带代码、配置、锁文件或生成产物。

## AI 生成内容审核

- 明确哪些内容由 AI 辅助生成或整理，使用范围是否只服务本仓库。
- Prompt、模型输出、生成图片、拆书结论或剧情文本不得默认视为事实；应检查是否需要人工确认、来源标记或质量复核。
- 生成正文、角色事实、世界设定、伏笔和状态回灌必须区分“已确认事实”和“待确认提案”。
- 公开文案要避免夸大自动化能力；无法稳定复现的能力不写成确定承诺。
- 如 AI 生成内容进入 README、公开文档、示例、截图说明或发布说明，应保持可维护、可追溯、可删除。

## 第三方材料审核

- 第三方代码、素材、字体、截图、数据集、模型输出、模型权重和参考文本都要记录来源、许可证和必要归属。
- 不确定许可证时，不合并为项目资产；可以改为文字描述、链接引用或自行重新制作。
- 外部图片和截图要确认是否有使用权，且不得泄露账号、API Key、用户稿件或私有项目数据。
- 保留 AGPL-3.0-only、NOTICE、FORK_NOTICE 等既有归属说明，不用维护者审核流程替代许可证义务。

## 测试与验证审核

- 优先运行与变更直接相关的验证命令；文档变更至少检查链接、标题和导航路径。
- UI 变更应有浏览器截图或手动路径记录，确认桌面和窄屏下入口可见、文字不溢出。
- 后端变更应覆盖成功路径、失败路径、权限或输入校验；任务链路还要检查重试和恢复。
- AI 链路应记录使用的 provider / model、是否真实调用、是否跳过扣费型验证，以及人工抽检结论。
- 如果无法运行测试，必须写明阻塞原因、未覆盖风险和建议补验方式。

## 合并前最小清单

| 检查项 | 通过标准 |
| --- | --- |
| 项目归属 | 变更只面向 Gonghan-Princess 维护的本仓库 |
| 用户入口 | README、公开文档或应用入口如受影响已同步 |
| 内部流程 | 不再引导到元项目 PR 或上游审核流程 |
| AI 内容 | AI 生成或整理内容已标注范围并完成必要人工复核 |
| 第三方材料 | 来源、许可证、归属和敏感信息已检查 |
| 测试结果 | 相关验证已运行，或明确说明无法验证的原因 |
| 并发改动 | 未回滚他人改动，冲突处按当前仓库状态处理 |

## 审核记录建议

维护者可以在任务回复、PR 描述、检查点文档或发布说明中记录：

- 本次变更影响的用户路径。
- 已检查的 UI / 后端 / AI 内容 / 第三方材料范围。
- 已运行的验证命令和结果。
- 未验证项、原因和后续补验建议。

记录目标是帮助 Gonghan-Princess 项目自己闭环，不是为了满足外部元项目流程。
